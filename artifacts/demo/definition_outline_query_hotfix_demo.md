# Definition Outline Query Hotfix Demo

## 目标

证明这轮 hotfix 做到三件事：

1. `X是什么？` 这类定义 / 提纲类问题，不再只靠 `X病` 连续词面命中选主依据
2. 六经病的 `X之为病 / X之病` 总纲句会被抬到主依据首位
3. `怎么办` 和 `条文是什么` 这两条边界没有被误伤

## 修复前后对照

### 修复前

- `太阳病是什么？`
  - `primary_evidence[0]` 是：`太阳病，项背强几几，无汗，恶风，葛根汤主之。`
  - 真正提纲句 `太阳之为病，脉浮，头项强痛而恶寒。` 只在补充依据里
- `阳明病是什么？`
  - `primary_evidence[0]` 是：`问曰：阳明病，外证云何？答曰……`
  - `阳明之为病，胃家实……` 没被抬成主依据

### 修复后

- `太阳病是什么？`
  - `primary_evidence[0]` 改为：`太阳之为病，脉浮，头项强痛而恶寒。`
  - `answer_text` 开头改成“书中对‘太阳病’的提纲性表述，可先看……”
- `阳明病是什么？`
  - `primary_evidence[0]` 改为：`阳明之为病，胃家实赵本有「是」字也。`
  - `answer_text` 不再围绕普通外证条文组织

## 最小验证

### 1. `太阳病是什么？`

- `answer_mode = strong`
- 主依据首位：
  - `太阳之为病，脉浮，头项强痛而恶寒。`
- 回答摘要：
  - 先点明“这是太阳病的提纲性表述”
  - 再给主依据条文
  - 其他中风 / 伤寒 / 治法条文退到补充依据

### 2. `阳明病是什么？`

- `answer_mode = strong`
- 主依据首位：
  - `阳明之为病，胃家实赵本有「是」字也。`

### 3. `少阳病是什么？`

- `answer_mode = strong`
- 主依据首位：
  - `少阳之赵本医统本皆有「为」字病，口苦、咽乾、目眩也。`
- 回答首句已清洗成：
  - `少阳之为病，口苦、咽乾、目眩也。`

### 4. `太阴病是什么？`

- `answer_mode = strong`
- 主依据首位：
  - `太阴之为病，腹满而吐，食不下，自利益甚，时腹自痛。若下之，必胸下结硬。`

### 5. `太阳病应该怎么办？`

- `answer_mode = strong`
- 仍按 general overview 路径输出“分情况整理”
- 未被误改成单句定义回答

### 6. `太阳病的条文是什么？`

- `answer_mode = strong`
- 仍按条文类直出回答
- 未被误改成“提纲句优先”的定义类模板

## 额外 spot check

- `少阴病是什么？`
  - 主依据首位：`少阴之为病，脉微细，但欲寐也。`
- `厥阴病是什么？`
  - 主依据首位：`厥阴之为病，消渴，气上撞心……`

## 结论

这轮修复后，定义 / 提纲类 query 已从“普通词面命中条文优先”切到“定义句 / 总纲句优先”；同时 general overview 与条文类路径仍保持原边界，没有被一并改坏。
