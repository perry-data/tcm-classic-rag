# Formula Effect Query Hotfix Demo

## 目标

证明这轮 hotfix 做到两件事：

1. `X方有什么作用` 不再默认答成方文 / 条文直出
2. 条文类与组成类边界仍然守住

## 修复前后对照

### 修复前

- `竹叶石膏汤方有什么作用？`
  - 开头仍是“根据主依据，直接相关的主条如下”
  - 优先吐方文与组成
  - 语境条文被淹没在后面

### 修复后

- `竹叶石膏汤方有什么作用？`
  - 开头改为“竹叶石膏汤在书中的直接使用语境，是‘伤寒解后，虚羸少气，气逆欲吐’”
  - 第二句用人话重述其主治倾向
  - 第三句再给条文依据
  - 方文只作为补充

## 最小验证

### 1. `竹叶石膏汤方有什么作用？`

- `answer_mode = strong`
- 实际回答摘要：
  - 先答直接使用语境：`伤寒解后，虚羸少气，气逆欲吐`
  - 再答主治倾向：偏向用于伤寒解后、虚羸少气、气逆欲吐这类情况
  - 再附依据条文与补充方文

### 2. `黄连汤方有什么作用？`

- `answer_mode = weak_with_review_notice`
- 实际回答摘要：
  - 未硬装 `strong`
  - 只按当前可核对材料，保守解释为：偏向用于`伤寒胸中有热，胃中有邪气，腹中痛，欲呕吐`这类情况
  - 明确提示当前未稳定找到正文层直接主治条文

### 3. `竹叶石膏汤方的条文是什么？`

- `answer_mode = strong`
- 实际回答摘要：
  - 继续按条文 / 方文直读输出
  - 未被误改成“作用解释类”

### 4. `竹叶石膏汤方由什么组成？`

- `answer_mode = strong`
- 实际回答摘要：
  - 继续按方文组成输出
  - 当前输出聚焦到本方组成行，未被误改成“作用解释类”

### 5. 边界样本

- 使用 query：`黄连汤方有什么作用？`
- 原因：
  - 当前能支持其直接使用语境的材料落在 `full:passages` 层
  - 因此系统只给 `weak_with_review_notice`
  - 没有把 review 层语境包装成正文级 `strong`

## 结论

这轮修复后，方剂作用类 query 已从“source_lookup 式条文直出”切到“先解释书中直接使用语境，再给依据”的结构；同时条文类、组成类仍保留各自回答边界，证据不足时也能保守落到 `weak_with_review_notice`。
