# Frontend Layout Refactor Before / After

## 对比摘要

| 项目 | 改前 | 改后 |
| --- | --- | --- |
| 页面主结构 | 顶部表单 + 下方结果面板 | 用户问题卡 + assistant response + 底部 composer |
| 输入区位置 | 页面顶部主区域 | 页面底部 |
| 用户问题显示 | 在 summary 中回显 | 单独作为 user message 展示 |
| 回答组织方式 | 多个 section 散落整页 | 一条 assistant response 统一承接 |
| 证据区位置 | 与回答并列成整页面板 | 当前回答下方的展开区 |
| review 材料 | 独立 section | 当前回答展开区内，默认折叠 |
| 样例入口 | 顶部显眼区域 | composer 内二级入口 |
| evidence 标题 | 可能直接复用正文首句 | 优先结构化简标题 + 相似性去重 |

## 典型阅读路径

### 改前

1. 先看到顶部大表单
2. 提交后下方出现多个不连续面板
3. 用户问题、回答、证据、引用分散在不同 section
4. 阅读时容易像在看“查询结果报表”

### 改后

1. 先看到当前用户问题
2. 紧接着看到当前系统回答
3. 回答下方再展开主依据 / 补充依据 / 核对材料 / 引用 / 改问建议
4. 继续提问时，直接回到底部 composer

## Evidence Card 具体前后差异

### 去重前

- `h3` 可能直接使用原始长标题
- `snippet` 再显示同一段正文
- 结果像：
  - 标题：`卫阳也荣阴也烧针益阳而损阴荣气微者谓阴虚也内经曰`
  - 正文：`卫阳也，荣阴也。烧针益阳而损阴。荣气微者，谓阴虚也……`

### 去重后

- `h3` 改为结构化简标题
- `snippet` 保留真实正文
- 结果像：
  - 标题：`辨脉法第一 · 注解 0016`
  - 正文：`卫阳也，荣阴也。烧针益阳而损阴。荣气微者，谓阴虚也……`

## 结论

本轮完成的不是“更好看的表单页”，而是：

- 信息架构从表单页切到单轮对话页
- 证据区从整页结果面板切到回答展开区
- 标题去重从“放任原始字段直出”切到“前端有规则地生成简标题”
