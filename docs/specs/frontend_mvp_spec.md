# 前端界面规格说明 (V1.0 毕业设计演示版)

- 文档版本：V1.0 (毕业设计演示版 / Defense-ready)
- 文档日期：2026-04-14
- 约束来源：`frontend/index.html` + `frontend/app.js` 真实实现
- 文档定位：定义 V1.0 版本的聊天式交互界面规范。

## 1. 界面设计目标

本版本的前端界面不再是简单的单次提问表单，而是一个支持会话管理的聊天式单页应用 (SPA)：

1. **会话流式体验**：模拟现代大模型聊天界面，支持问题与回答的连续展示。
2. **历史记录管理**：支持从侧边栏查看、切换和删除历史会话。
3. **证据分区展示**：清晰呈现主证据、辅助依据、参考资料及引用。
4. **Markdown 渲染**：支持回答文本中的格式化显示。

## 2. 核心布局 (Layout)

界面采用双栏布局：

- **左侧边栏 (Sidebar)**：
  - “新建对话”按钮。
  - 历史会话列表（按时间排序）。
  - 会话搜索框。
- **右侧主面板 (Main Panel)**：
  - **顶部工具栏**：显示当前会话标题、状态提示。
  - **消息流 (Message Feed)**：滚动展示用户提问与系统回答。
  - **底部输入区 (Composer)**：多行文本输入框、发送按钮、示例问题抽屉。

## 3. 渲染规则

### 3.1 消息气泡渲染
- **用户消息**：右侧展示，简洁背景。
- **系统回复**：左侧展示，包含：
  - **回答正文**：经过 Markdown 渲染的文本。
  - **依据分区**：`主证据`、`辅助依据`、`参考资料` 分块展示。
  - **引用区**：展示具体的条文编号与出处。
  - **改问建议**：以点击式 Chip 形式展示在底部。

### 3.2 模式响应
- **Strong**：正常展示所有区域。
- **Weak**：显式展示 `review_notice`（黄色背景提醒），主证据区隐藏。
- **Refuse**：显著展示拒答原因，隐藏证据区。

## 4. 交互逻辑

1. **发送消息**：
   - 禁用发送按钮，显示 Loading。
   - 调用 `POST /api/v1/conversations/{id}/messages` 或 `POST /api/v1/answers`。
   - 滚动到底部。
2. **切换会话**：
   - 从侧边栏点击会话。
   - 清空当前消息流，加载目标会话的历史记录。
3. **示例问题填充**：
   - 点击示例 Chip，仅填充输入框，不自动发送。

## 5. 技术约束

- **框架**：Vanilla JS (ES Modules) + CSS Grid/Flexbox。
- **部署**：由 Python 后端提供同源静态资源服务。
- **通信**：标准的 JSON API。

## 6. 与开题报告的对齐

虽然未采用 React 技术栈，但 V1.0 界面已完全实现了开题报告要求的“友好 Web 交互”与“证据溯源展示”，并额外提供了超出开题范围的“历史会话管理”功能。
