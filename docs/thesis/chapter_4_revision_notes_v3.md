# 第4章 v3 扩写修订说明

## 1. 扩写与优化方向
本版本（v3）在 v2 事实骨架的基础上进行了系统性的扩写，使其在篇幅和行文深度上达到毕业论文正文的标准。具体扩写重点包括：
- **4.1 章节**：补充说明了系统的具体适用场景（中医经典理论学习、文献溯源、方证对比），并进一步强化了“非诊断系统”的系统边界描述。
- **4.2 章节**：对架构模型中各层级的职责和生命周期进行了丰富，详细描述了数据在表示层、应用接口层、逻辑编排层及数据模型层中的流转逻辑与相互调用关系。
- **4.3 章节**：详细论述了统一视图（Unified View）在管理不同类型检索记录时的作用，补充了安全数据集（Safe Dataset）剔除边缘数据的离线预处理意义，并解释了 FTS5 三元组分词与 FAISS 稠密向量在应对古文术语时的设计必要性。
- **4.4 章节**：在在线问答流程中补充了意图识别拦截无效提问、证据排序及答案生成的详细步骤拆解。
- **4.5 章节**：详细解释了为什么 BM25 和余弦相似度由于量纲不同需要 RRF 融合，以及双编码器（Bi-encoder）的局限性为何需要引入交叉编码器（Reranker）的二次提纯。
- **4.6 章节**：丰富了主依据、补充依据及核对材料的功能定位，并对 Strong/Weak/Refuse 三种回答模式的触发条件、风险控制机制及其工程价值进行了深度阐述。
- **4.7 章节**：对 Prompt 模板中基于槽位的注入技术、前后端通过 `citations` 数组进行的结构化映射，以及前端实现“以文寻证”闭环交互的 DOM 渲染逻辑进行了详细说明。
- **4.8 章节**：客观分析了放弃现代前端框架（如 React）转而使用原生技术的工程考量：减少虚拟 DOM 在大段文本和跨区高亮时的复杂抽象，以及降低内置服务器的部署成本，避免了对性能的过度宣扬。
- **4.9 章节**：在可评测性设计中，进一步阐述了 Hit@K 衡量召回敏锐度、Mode Match 评估门控准确性、Citation Pass 验证防幻觉能力的具体作用。

## 2. 移除与修正的风险表述
- **移除虚构特性**：全文未新增任何未经验证的功能（如：上下文感知检索、多维向量摘要、SSE 流式推送、异体字库映射等均已被剔除）。
- **去浮夸用语**：删除了“最高性能响应”、“创新性地”等过度拔高的修辞，将行文风格调整为客观保守的学术论述。
- **术语规范化**：将表4-2中的具体数字阈值 `0.65 / 0.4` 修改为客观的“系统设定高阈值/系统设定低阈值”；将“权威注解”修正为“注解记录”。
- **客观的标标注说明**：明确可评测性中为“标准化的答案标注”，不涉及未证实的“中医专家”头衔。
