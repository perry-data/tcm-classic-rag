# 第4章 v4 微修封板版说明

## 1. 弱化的绝对化表述
在本次 v4 版本修订中，对 v3 中的一些绝对化和夸大词汇进行了全面审视与降级处理，以保障学术论文的严谨性：
- 将“从数据源头切断模型自主编造知识的可能”修正为“从数据与流程层面降低模型自主编造知识的风险”。
- 将“原文与注解记录处于绝对的支配地位”修正为“原文与注解记录在回答生成中具有优先约束作用”。
- 将“此时，系统彻底切断 LLM 的生成流程”修正为“此时，系统不再调用 LLM 生成解释性回答”。
- 将“彻底阻断模型在无证据时的编造行为”（表 4-2）修正为“避免模型在无有效证据时继续生成解释性回答”。
- 移除了类似“极高”、“最高”、“完全”、“创新性地”等缺乏实验数据明确支撑的修辞，如“具有极高的精确匹配召回率”改为“具有较高的精确匹配召回率”，“量纲上完全不同”改为“量纲上存在差异”。

## 2. 优化与修正的其他细节
- **Safe Dataset 表述优化**：将原本“剔除了一些高度模糊或可能引发系统拒答误判的边缘数据”修正为更严谨的学术表述：“对高风险、低置信度或不适合直接作为主证据的材料进行了降级、隔离或过滤处理”。
- **风险词汇筛查**：经过全文核对，确认文本中不再包含“专家标注”、“SSE”、“上下文感知检索”、“多维向量”以及“异体字库”等未在代码仓库中核验证实的高风险技术主张。

## 3. 核心设计要点（已核验保留）
本章依然保留并详述了系统经过代码事实核验的核心设计：
- 系统的非辅助诊断边界设定。
- 采用原生 HTML/CSS/Vanilla JS 驱动前端。
- 依赖 `/api/v1/answers` 和会话持久化 (`ConversationStore`)。
- 混合检索（SQLite FTS5 + FAISS）及 RRF、BGE Reranker。
- AnswerAssembler 控制的 Evidence Gating（证据门控机制）。
- 证据的三级划分（Primary / Secondary / Review）和回答模式的三级控制（Strong / Weak_with_review_notice / Refuse）。
- 基于 citations 数组的前端引用联动。
- 包含 150 条用例的 Goldset 黄金测试集与 Evaluator v2 评测设计。

本封板版在保证行文通畅与设计说明深度的同时，去除了所有的风险表述，是稳定、可答辩且严格对齐仓库真实状态的版本。
