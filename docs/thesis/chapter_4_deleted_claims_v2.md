# 第4章 删除/修改记录 (v2)

本文件记录了在 v2 版本重写中，针对 v1 版本及开题愿景中所有高风险、无证据表述的处理情况。

| 删除/修改前表述 | 处理方式 | 原因 | 是否需要人工确认 |
|---|---|---|---|
| 古医药学异体字映射库 | 修改为“核心方药名映射” | `assembler.py` 中仅有 3 组元组映射，称作“库”属于夸大。 | 否 |
| 40,000+ 语义片段 | 修改为“4,000 余条统一检索记录” | `database_counts.json` 显示 Chunks 仅 583 个，Unified 视图记录为 4280 条。 | 否 |
| SSE / 流式推送证据 | 改为“后端支持流式接口路径” | 后端有路径但前端 `app.js` 仍使用标准 `fetch`。论文正文 4.8.2 中已修正表述。 | 是 |
| 逻辑锚点切分法 | 删除 | 代码中未发现特定逻辑锚点，仅为基于条文 ID 的常规 Chunking。 | 否 |
| 独立摘要向量/多维向量 | 删除 | `hybrid.py` 中仅有单一的向量编码逻辑，无多维摘要向量设计。 | 否 |
| 上下文感知检索 | 删除 | `assembler.py` 的 `assemble` 方法目前仅处理单次 Query，无上下文拼接逻辑。 | 否 |
| 专家/未参与人员标注 | 修改为“标准化标注” | 无证据文件证明标注人员身份，出于论文保守性考虑进行降级表述。 | 是 |
| 彻底杜绝幻觉 | 修改为“降低幻觉风险” | “彻底杜绝”属于绝对化描述，不符合学术严谨性。 | 否 |
| 秒级防幻觉 | 删除 | 属于非学术性宣传语。 | 否 |
| 洗沙取金 / 证据保管员 | 删除 | 属于过度人格化或非学术修辞。 | 否 |
| 极高鲁棒性 | 删除 | 属于评价性词汇，缺乏实验数据支持。 | 否 |
| 多会话并行支持 | 修改为“支持多会话切换” | 真实代码支持会话 ID 切换，但“并行”表述可能引起理解偏差。 | 否 |
| 逻辑统一、视觉保留 | 删除 | 属于设计口诀，无直接工程证据。 | 否 |
| 独立方剂名称向量 | 删除 | 检索引擎目前统一使用 BGE 对所有文本进行向量化，无独立方剂向量分支。 | 否 |
