# 评估实施计划 v1

- 文档版本：v1
- 文档日期：2026-04-06
- 文档定位：定义本轮先做哪些评估，以及如何从 seed goldset 扩展到完整测试集

## 1. 本轮先做什么

本轮先建立“评估底座”，而不是继续扩功能。

当前落地项只有五件事：

1. 固定评估规格：明确评估目标、题型、维度和边界。
2. 固定标注规则：明确标准依据、证据片段和引用核对怎么写。
3. 固定数据结构：建立可扩写的 `goldset_schema_v1.json`。
4. 固定 seed 样本：建立一批高质量、可追溯、可复用的 seed questions。
5. 固定最小验证：确认 seed goldset 的 JSON 结构、题型覆盖和当前系统消费路径。

## 2. 为什么本轮先做 seed goldset

本轮不直接硬凑 200–250 条，原因有四点：

1. 当前最缺的不是“题目数量”，而是统一 schema、统一标注规则和统一判分口径。
2. 如果在规则未冻结前直接扩到 200–250 条，后续很容易出现字段返工、题型混乱和引用标准不一致。
3. 当前正式系统刚完成 hybrid retrieval、evidence gating 和 answer assembler 的冻结，更适合先用一批代表性题目把评估链路立稳。
4. seed goldset 已足够覆盖当前主要问题类型，可先为论文第 4 章、答辩演示和功能回归提供正式基线。

换句话说，本轮先做的是“把尺子做准”，而不是“先把卷子堆满”。

## 3. 当前 seed 范围

本轮 seed goldset v1 采用 9 条高质量样本，覆盖五类主问题：

| 题型 | 当前 seed 目的 | 当前数量 |
| --- | --- | --- |
| 条文/出处类 | 作为稳定 `strong` 基线 | 1 |
| 含义解释类 | 验证 `weak_with_review_notice` 与主依据为空 | 1 |
| 泛问/总括类 | 覆盖强回答与降级回答两种情况 | 3 |
| 比较类 | 覆盖可比较与需降级比较两种情况 | 2 |
| 无证据拒答类 | 覆盖书外拒答与功能边界拒答 | 2 |

这些题目优先选自当前已经存在的 frozen examples / smoke examples，目的是让 seed 集与正式系统现状保持一致。

## 4. 本轮评估如何执行

当前 v1 评估采用“已有系统 replay + 人工核对”的最小方案：

1. 用 `goldset_v1_seed.json` 保存标准问题与 gold 依据。
2. 用现有 answer assembler / API 重放这些 query。
3. 先检查 `answer_mode` 是否符合 `expected_mode`。
4. 再检查 citations 是否落在 `gold_record_ids` 或同一 `gold_passage_ids`。
5. 最后人工判断是否存在无证据强答、错引强答或越级引用。

当前仍未加入独立 evaluator runner，因此本轮 smoke checks 的目标是“建立正式可审查基线”，不是“产出最终自动化分数”。

## 5. 后续如何扩展到 200–250 条

后续扩容建议分三步走：

### 阶段 A：规则稳固

先把当前 9 条 seed 用于：

1. 对齐题型定义。
2. 对齐 `expected_mode` 判定。
3. 对齐 gold passage 与 citation 核对规则。

当 seed 标注口径稳定后，再进入批量扩写。

### 阶段 B：扩到 60–80 条

建议优先按题型补齐：

1. 条文/出处类：覆盖更多方名、章节与出处追问。
2. 含义解释类：覆盖语句释义、术语释义和注解依赖场景。
3. 泛问/总括类：覆盖太阳、阳明、少阴等不同主题。
4. 比较类：覆盖“区别 / 异同 / 多了什么 / 少了什么”。
5. 拒答类：覆盖书外问题、优劣判断和超宽泛问法。

### 阶段 C：扩到 200–250 条

完整集建议采用近似配额，而不是平均分配：

| 题型 | 建议目标 |
| --- | --- |
| 条文/出处类 | 60 |
| 含义解释类 | 45 |
| 泛问/总括类 | 40 |
| 比较类 | 30 |
| 无证据拒答类 | 35 |

按上述配额，完整集约为 210 条，处于开题报告承诺的 200–250 条范围内。

## 6. 扩写原则

后续扩写必须遵守以下原则：

1. 先扩题型覆盖，再扩题目数量。
2. 先扩可稳定标注的题，再扩争议较大的题。
3. 每条样本都必须能回指明确条文或明确拒答边界。
4. 不允许为了凑量而加入无法写清 gold 依据的题。
5. 扩写过程中继续保持单书《伤寒论》边界，不提前混入多书样本。

## 7. 本轮完成后的正式意义

本轮完成后，项目已经具备：

1. 可写入论文第 4 章的正式评估口径。
2. 可用于答辩演示的 seed 测试集。
3. 可供后续 200–250 条扩写直接复用的 schema 和标注规则。

因此，本轮不追求“样本做满”，而是追求“评估底座立稳”。
