# ambiguous annotation 人工抽样复核报告 v1

## 1. 抽样构成

本轮共复核 60 条样本，分层如下：

| 分层 | 数量 | 说明 |
| --- | ---: | --- |
| A | 20 | 高风险 / 被移除 / 被降级的 `main_passages` |
| B | 20 | annotation / annotation 挂接不稳样本 |
| C | 10 | 问题导向样例：发汗药 / 太阳病 / 竹叶石膏汤方作用 |
| D | 10 | 当前 safe 中保留且相对稳定的对照样本 |

按当前 safe 状态统计：

- `removed`: 10

按首轮复核分级统计：

| 统计口径 | L1 | L2 | L3 | L4 |
| --- | ---: | ---: | ---: | ---: |

另外做了原始 zip 的点状回溯，用来确认关键样本并非只是在结构化层“看起来可疑”：

- `1《卷一》` 的 `73 -> 74`、`77 -> 78` 排布与验收报告一致，支持“挂到前一条”的错挂判断。
- `2《卷二》` 的第 120 条原文中，`桂枝汤者，发汗药也` 确实独立出现，说明它不是导出脚本拼接假象。
- `7《卷七》` 的 `63 / 64 / 65` 顺序清晰，支持“竹叶石膏汤主条 + 解释条 + 方题”这一组问题样例判断。

## 2. 各类样本的初步观察

### 2.1 A 类：高风险 `main_passages`

观察结果不是“全该恢复”，也不是“全都该继续封死”，而是明显混合：

- 有一批条目虽然被 `ambiguous` 命中，但文本本体其实完整，且与相邻稳定条文存在清晰近邻关系。
- 也有一批条目虽然可读，但解释链太长、外引太多、上下文依赖很重，更适合作为 review。
- 短文本降级里存在一类“短但完整”的定义句和方证句，这类样本显示 safe 的短句规则有局部过保守迹象。

最明显的正样本包括：

- `ZJSHL-CH-008-P-0201`：`太阳病，发热而渴，不恶寒者，为温病。`
- `ZJSHL-CH-014-P-0021`：`少阴之为病，脉微细，但欲寐也。`
- `ZJSHL-CH-015-P-0251`：`手足厥寒，脉细欲绝者，当归四逆汤主之。`
- `ZJSHL-CH-010-P-0168`：`伤寒脉结代，心动悸，炙甘草汤主之。`

这些都不是“长篇解释型材料”，而是短而完整、适合研读型 RAG 的高信息密度条目。

### 2.2 B 类：annotation 与挂接不稳样本

annotation 层呈现出很清楚的“两极化”：

- 一端是已确认错挂或高度疑似错挂的样本。尤其是 `LINK-00033/34/36/37/38/39` 这组，连续表现出“挂到前一条”的偏移模式。
- 另一端是文本本身解释力不错、而且与当前 target 贴合度较高的样本，例如 `ZJSHL-CH-008-P-0192/0194/0196/0216` 这组。

这意味着：

- `annotation_links` 作为自动挂接层，仍然不能视为可靠。
- 但 annotation 原文作为“未启用自动挂接的辅助材料”，并非完全没有受控放开空间。

### 2.3 C 类：问题导向样例

`发汗药`

- 问题最关键的解释性条目之一 `ZJSHL-CH-006-P-0120` 当前被 safe 移除。
- 同主题还存在 `ZJSHL-CH-006-P-0127` 这类使用说明型条目，可补召回但不宜主证据化。
- 说明 safe 对“主题性解释条”存在局部过保守。

`太阳病是什么`

- safe 内已有高价值定义条 `ZJSHL-CH-008-P-0191`，并非完全答不出来。
- 但被移除的 `ZJSHL-CH-008-P-0201` 和稳定 annotation `ZJSHL-CH-008-P-0192/0194/0196/0216` 能显著补足定义边界和分支解释。
- 因此，这个问题更像“safe 基线可答，但解释层仍偏薄”。

`竹叶石膏汤方有什么作用`

- safe 内主条 `ZJSHL-CH-017-P-0063` 已可承担核心依据。
- 被移除的 `ZJSHL-CH-017-P-0064` 正好提供“津液不足、馀热未尽、调胃散热”的解释性补足。
- 这说明并非所有 removed 条都该恢复，但确实存在“很适合回到 recall / review”的局部补充解释。

### 2.4 D 类：safe 对照样本

对照样本显示 safe 本身并不是全面失效的：

- 大量正文主条、方题、方后法在 safe 中保持稳定。
- 尤其 `辨太阳病脉证并治` 与 `辨太阳病脉证并治第六` 里的部分方条，结构完整、适合当前主链继续使用。

因此，本轮不支持“safe 整体失败”的判断；问题主要集中在特定被移除条、短句降级规则，以及 annotation 自动挂接层。

## 3. 错杀迹象是否存在

结论：存在，而且是“局部明显存在”，但不是“整库都被错杀”。

支撑点：

1. 在 20 条 A 类高风险主条里，有 4 条进入 L1、8 条进入 L2。
2. 被 short demotion 的条目中，出现了多条“短但完整”的定义/方证句。
3. 查询导向样例里，`发汗药` 与 `竹叶石膏汤方作用` 的关键补充解释，确实落在 removed 条中。

因此，safe 当前并非“整体过保守”，但对以下两类内容明显偏严：

- `ambiguous` 命中但语义完整、且与邻接条文关系清晰的解释条。
- 短但完整的定义句、方证句、主治句。

## 4. annotation 挂接是整体不稳，还是局部可放开

结论：自动挂接层整体仍不稳，但 annotation 原文存在局部可放开空间。

更具体地说：

1. `annotation_links` 作为自动挂接层，仍应视为“整体不稳”。
2. 已确认错挂与启发式疑似错挂共同说明，不能把当前 link 层当成可直接启用的证据链。
3. 但 annotation 文本本身并非一律不可用。B 类 20 条里有 9 条进入 L2，说明存在一批可作为 `raw annotation` 受控回放的样本。

所以正确结论不是：

- 不是“所有 annotation 都该放开”；
- 也不是“所有 annotation 都该继续封死”；
- 而是“继续禁用自动挂接，只讨论局部 raw annotation 的 recall / secondary 受控放开”。

## 5. 是否建议进入下一轮“受控回放实验”

结论：建议，而且值得做，但范围必须小。

建议下一轮分两条实验轨：

### 5.1 轨道一：`main_passages` 小范围受控回放

范围建议：

- 先只回放本轮 L1 + 部分 L2 的 `main_passages`。
- 优先选“短但完整”的定义句 / 方证句，以及与问题样例直接相关的 removed 解释条。
- 向量索引只试点少量语义自足条目，不扩大到全部 removed 内容。

优先候选示例：

- L1：`ZJSHL-CH-008-P-0201`、`ZJSHL-CH-014-P-0021`、`ZJSHL-CH-015-P-0251`、`ZJSHL-CH-010-P-0168`、`ZJSHL-CH-017-P-0063`
- L2：`ZJSHL-CH-006-P-0120`、`ZJSHL-CH-006-P-0127`、`ZJSHL-CH-017-P-0064`、`ZJSHL-CH-006-P-0102`、`ZJSHL-CH-006-P-0109`

### 5.2 轨道二：raw annotation 受控回放

范围建议：

- 不恢复 `annotation_links`。
- 只让少量 L2 annotation 以“注解原文，未启用自动挂接”的口径参与 `recall / secondary`。
- 暂不纳入向量索引。

优先候选示例：

- `ZJSHL-CH-008-P-0192`
- `ZJSHL-CH-008-P-0194`
- `ZJSHL-CH-008-P-0196`
- `ZJSHL-CH-008-P-0216`
- `ZJSHL-CH-003-P-0053`
- `ZJSHL-CH-003-P-0098`

## 6. 明确结论：哪些内容可以考虑放回到哪一层

### 6.1 可考虑回到 `primary` 候选池的内容

- short demotion 中的完整定义句、方证句、主治句
- 少数 removed 但语义完整、并经人工确认边界无误的定义型条文

本轮对应 L1。

### 6.2 可考虑回到 `recall / vector` 的内容

- removed 但完整、与问题样例强相关的解释性主条
- short demotion 中的规则句、概括句

本轮对应的 `vector` 候选只限于 `main_passages` 的部分 L2，不包含 annotation。

### 6.3 可考虑回到 `recall / secondary` 的 annotation

- 只限文本本身解释力较强、且未见明显错挂信号的 raw annotation
- 展示时必须显式标注“注解原文，未启用自动挂接”

### 6.4 只保留在 `review` 的内容

- 长解释但上下文依赖重的 ambiguous 主条
- 疑似错挂但文本仍有核对价值的 annotation
- 只适合人工研读、但不适合线上常规召回的短句

### 6.5 继续排除的内容

- 已确认错挂且高度摘要化的 annotation
- 一句式病机摘要、脏绝摘要、明显碎片化短句
- 任何依赖当前错误 link 才看起来“像证据”的内容

## 7. 本轮不建议做什么

1. 不建议恢复 `annotation_links`。
2. 不建议把 annotation 全量放进向量索引。
3. 不建议把 `ambiguous_passages` 相关条目整批回灌主检索层。
4. 不建议直接修改 answer assembler 来“特判修复”这三个问题。
5. 不建议因为发现局部错杀，就推翻 safe 基线本身。

## 8. 四个问题的直接回答

1. safe 是否明显过保守  
   结论：存在局部过保守，主要体现在“完整短句被降级”和“少量 query-critical 解释条被整体移除”；但 safe 不是整体失败。

2. 是否存在高风险 `main_passages` 被错杀  
   结论：存在。A 类样本中已有 4 条 L1、8 条 L2，说明确有一批值得受控回放。

3. annotation 是否有局部可放开空间  
   结论：有，但仅限 raw annotation 的 recall / secondary；自动挂接层仍不应放开。

4. 下一轮是否值得做“受控回放实验”  
   结论：值得，而且建议先做小范围、分轨道实验，不做大规模放开。
