# ambiguous annotation 人工抽样复核计划 v1

## 1. 本轮复核目标

本轮只做一轮最小闭环的“人工抽样复核”，回答四个问题：

1. safe 模式是否存在局部过保守，导致可用内容被错杀。
2. 高风险 `main_passages` 中是否存在值得恢复的条目。
3. annotation 挂接问题究竟是“整体不稳”，还是“存在一部分可受控放开的条目”。
4. 后续是否值得把部分内容放回 `primary / recall / vector / review` 各层。

本轮输出只给“复核事实 + 分级建议”，不直接改检索主链，不直接放开 `annotation_links`，也不重建索引。

## 2. 输入边界

本轮只使用已冻结底座与必要回溯来源：

- `data/processed/zjshl_dataset_v2/`
- `dist/zjshl_dataset_v2_mvp_safe.zip`
- `data/raw/《注解伤寒论》.zip`
- 既有验收与分层策略文档

本轮明确不做：

- 不改 `annotation_links` 启用状态。
- 不改主检索逻辑、rerank、evidence gating、payload contract。
- 不改前端与 evaluator。
- 不重开原始 15 个 md 的解析工程。

## 3. 抽样分层策略

总样本量控制为 60 条，采用“问题导向 + 分层抽样”，而不是纯随机抽样。

| 分层 | 数量 | 抽样目标 | 说明 |
| --- | ---: | --- | --- |
| A | 20 | 高风险 / 被移除 / 被降级的 `main_passages` | 覆盖 `ambiguous` 被移除条与短文本被降级条，观察 safe 是否错杀。 |
| B | 20 | annotation / annotation 挂接不稳样本 | 覆盖 6 条已确认错挂、启发式疑似错挂样本、相对稳定的 annotation 对照样本。 |
| C | 10 | 问题导向样例 | 纳入“发汗药 / 太阳病 / 竹叶石膏汤方作用”三个问题的候选支持条。 |
| D | 10 | 当前 safe 中保留且相对稳定的对照样本 | 作为 safe 基线校准，避免把所有问题都归因为 safe 过严。 |

### 3.1 A 类样本的再分层

- `removed + ambiguous`：优先抽取语义较完整、与相邻条文存在明显近邻关系的条目。
- `downgraded + short`：同时抽取“短但完整”的定义/方证句，和“短且碎片化”的高风险句。

### 3.2 B 类样本的再分层

- 已确认错挂：直接纳入，观察“自动挂接错误”是否具有连续偏移模式。
- 启发式疑似错挂：沿用验收脚本的相邻主条相似度口径，观察是否存在“挂到前一条”的批量偏移。
- 稳定 annotation 对照：选取未被验收报告点名、且正文-注解内容贴合度高的样本，观察是否存在“只按注解原文受控放开”的空间。

### 3.3 C 类问题导向样例

三组问题样例必须进入抽样：

- `什么是发汗药`
- `太阳病是什么`
- `竹叶石膏汤方有什么作用`

问题导向样例既看“当前 safe 有没有稳定主条”，也看“被移除 / 被禁用的内容里是否藏着关键补充解释”。

### 3.4 D 类对照样本

对照样本只选当前 safe 中保留、文本较完整、结构较稳定的 `main_passages`，用于校准：

- safe 并非对所有长条或方条都过保守；
- 真正的问题更可能集中在 `ambiguous` 命中、短句降级和 annotation 挂接层。

## 4. 复核判定标准

每条样本至少从以下五个角度判断：

1. 文本本体是否完整。
2. 语义是否自足。
3. 与锚点 / 条文挂接是否稳定。
4. 是否容易造成错引、串条、伪定义、伪作用解释。
5. 是否适合古籍研读型 RAG 的使用场景。

### 4.1 main_passages 判定重点

- 若是 `ambiguous` 被移除条：重点看它究竟是“完整可读的解释条”，还是“依赖上下文的半句、引文、过度推演”。
- 若是短句被降级条：重点看它是“短但完整”的定义/方证句，还是“短且脆弱”的碎片句。

### 4.2 annotation 判定重点

- annotation 文本本身是否具有独立解释价值。
- 当前 link 是否已确认错挂，或存在明显偏移信号。
- 即便 link 不稳，annotation 原文是否仍可作为“未启用自动挂接的辅助材料”参与受控召回。

## 5. 四级放回建议口径

| Level | 含义 | 可进入层级 | 不可做什么 |
| --- | --- | --- | --- |
| L1 | 可恢复为较强候选证据 | 可讨论进入 `primary` 候选池，也可进入 `recall / vector` | 本轮不直接上线放开 |
| L2 | 不宜做主依据，但可恢复到召回层 / 向量层 | `recall` 必须可控；`vector` 只给语义自足且风险可控的条目 | 不直接抬成主依据 |
| L3 | 仅可作为 review / risk material | `review` | 不进入常规召回和主证据链 |
| L4 | 继续排除 | 无 | 不放回任何线上主路径 |

### 5.1 关于 vector 的额外约束

- 只有当文本语义自足、且不严重依赖正确挂点时，才讨论进入 `vector`。
- annotation 即便进入 L2，本轮默认也只讨论 `recall / secondary`，不直接建议入向量索引。

## 6. 复核执行方式

本轮采用“结构化样本导出 + 人工首轮判定”的方式：

- 用 `scripts/export_ambiguous_annotation_review_sample.py` 导出 manifest 与 review sheet。
- 以 `data/processed/zjshl_dataset_v2/` 为主工作底座。
- 对关键样本用 `data/raw/《注解伤寒论》.zip` 做点状回溯，不重跑解析。
- 复核结果先落在 `artifacts/review/`，再汇总进入报告。

## 7. 风险说明

- 抽样复核只能回答“是否存在局部可放开空间”，不能替代整库重验收。
- 对 annotation 的任何正向结论，只适用于“raw annotation 受控使用”，不等于可以恢复 `annotation_links`。
- 对 `main_passages` 的正向结论，只是“候选放回建议”，不等于可以直接回灌主检索链。
- 本轮报告只支撑下一轮“受控回放实验”，不支撑“一刀切全面放开”。
