# 分层启用样例流转说明

## 场景 A：命中高置信主条

### 问题示例

用户提问：`桂枝汤方在书中如何表述？`

### 召回阶段

1. 优先命中 safe `chunks`。
2. 命中片段回指到 safe `main_passages(retrieval_primary=true)`。
3. 若同时命中若干注解片段，只保留为候选辅助材料，不影响主证据判断。

### 证据筛选阶段

1. 选出 1 到 3 条 Level A 主条作为 `primary_evidence`。
2. 若有直接解释药性或方义的 raw `annotations`，可放入 `secondary_evidence`。
3. 不使用 `annotation_links` 自动宣称“这条注解就是该正文的权威解释”。

### 回答阶段

- 允许输出标准回答。
- 回答主体基于 Level A。
- 若附带注解，只能写成“可参考的注解原文片段”，不能替代正文主依据。

### 页面展示建议

- 主依据区：
  展示 `main_passages` 原文。

- 命中片段区：
  可展示 `chunks` 命中片段作为检索预览。

- 参考说明区：
  如有需要，展示注解原文，并标 `注解参考（未启用自动挂接）`。

## 场景 B：只有注解或低置信材料命中

### 问题示例

用户提问：`“肺先绝也”在注解里怎么解释？`

### 召回阶段

1. 可能先命中 raw `annotations`。
2. 同时尝试召回 safe `main_passages` 与 safe `chunks`。
3. 若没有足够的 Level A 命中，再看是否存在同章或近邻正文作为背景。

### 证据筛选阶段

1. 若没有 Level A，只能将注解放入 `secondary_evidence`。
2. 若命中的还包含 `ambiguous_passages` 关联内容，则这些材料只能归入 `risk_materials`。
3. 此时不得输出“已证实的正文依据是……”这类强表述。

### 回答阶段

- 允许回答，但必须是弱回答。
- 必须显式提示：
  `当前主要命中注解或风险材料，正文强证据不足，以下内容需核对。`

- 如果注解内容内部也不稳定，或只命中 `ambiguous_passages`，则应直接拒答或要求用户改问更具体条文。

### 页面展示建议

- 不显示“主依据”标题，或显示为空状态。
- 显示“辅助材料”区：
  放 raw `annotations`。

- 如有风险材料，再单独显示“待核对材料”区：
  放 Level C 内容，并带 `低置信度 / 待核对` 标签。

## 场景 C：召回很多，但强证据不足

### 问题示例

用户提问：`阴阳交易在人变病中的含义是什么？`

### 召回阶段

1. 可能从 safe `chunks`、safe `main_passages`、raw `passages`、raw `annotations` 中同时召回大量候选。
2. 命中结果分散在多个章节或多个语义片段中，未形成稳定的 Level A 主证据簇。

### 证据筛选阶段

1. 若 Level A 命中分散且无法形成清晰主依据，不应强行拼成“完整解释”。
2. 可保留少量同主题的 Level B / Level C 片段作为背景参考。
3. 只要主证据不足，就不能进入强回答模式。

### 回答阶段

可采用两种处理方式：

1. 半回答
   适用于存在少量相关 Level B 材料，但不足以形成稳定 Level A 结论时。
   表述方式应为：
   `从当前召回结果看，该问题涉及若干相关表述，但能够直接作为主依据的条文不足，以下仅给出待核对的归纳。`

2. 拒答
   适用于只命中风险材料、或候选彼此冲突、或无法收敛到明确条文时。
   表述方式应为：
   `当前未找到足以支撑回答的主依据，建议缩小到具体条文、方名或句段再查询。`

### 页面展示建议

- 主依据区：
  若没有 Level A，则不展示或显示“主依据不足”。

- 参考材料区：
  展示少量 Level B。

- 待核对材料区：
  若系统决定保留 Level C 命中，则单独展示，且默认折叠。

## 补充说明：前台分区原则

无论哪种场景，页面层都应始终区分三类区域：

1. `主依据`
   只放 Level A。

2. `辅助说明`
   只放 Level B。

3. `待核对材料`
   只放 Level C。

这样做的目的不是让页面更复杂，而是防止系统把不同可信度的数据混成一层，导致用户误判证据强度。
