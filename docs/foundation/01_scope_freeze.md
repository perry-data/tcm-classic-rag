# 毕业设计 V1.0 范围冻结文档 (毕业设计演示版)

- 文档版本：V1.0 (毕业设计演示版 / Defense-ready)
- 文档日期：2026-04-14

## 1. 项目目标

本项目实现了一个面向《伤寒论》文本研读场景的智能支持系统，用于支持用户围绕条文内容进行语义检索、提问回答、证据分层与历史回溯。系统采用单体 Web 应用形态，已完成从“自然语言提问”到“返回带证据的 LLM 润色回答”的 V1.0 闭环。

本项目的核心价值在于验证“基于检索增强生成 (RAG) 与证据溯源的经典文本研读支持”在单书场景下的可行性，并解决了大模型在专业领域的知识幻觉问题。

## 2. 核心功能 (已落地)

### 2.1 单书语料范围
- 系统语料严格限定为《伤寒论》原文及核心注解。
- 确保了检索与证据的权威性与集中度。

### 2.2 混合检索引擎
- 实现了 FTS5 稀疏检索与 FAISS 稠密检索的混合召回。
- 跨越了现代汉语提问与文言文条文之间的语义鸿沟。

### 2.3 证据分层与门控 (Evidence Gating)
- 建立了 `Primary / Secondary / Review` 三级证据槽位。
- 实现了“证据不足则弱回答或拒答”的风险控制逻辑。

### 2.4 LLM 驱动的回答生成
- 接入了通义千问 (Qwen-plus) 大模型。
- LLM 仅负责在证据约束下润色回答文本，确保“言之有据”。

### 2.5 交互与会话管理
- 实现了聊天式 Web 界面。
- 支持历史会话的持久化存储与查看。

## 3. 明确不做的内容 (V1.0 边界)

- 多书扩展（如《金匮要略》等）。
- 医疗诊断、辨证论治、处方推荐等诊疗能力。
- 生产级高并发、复杂微服务架构。
- 复杂知识图谱或多轮复杂推理。

## 4. 成功标准 (答辩验收)

- 用户能在 Web 界面输入自然语言问题。
- 系统能返回与问题相关的条文证据分区展示。
- 系统能生成逻辑清晰、出处明确的润色回答。
- 当证据缺失时，系统能明确提示或拒答。
- 整个流程在演示中能充分体现其“研读支持”与“非诊疗”的学术定位。
