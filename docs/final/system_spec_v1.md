# 基于检索增强生成的中医经典研读支持系统系统规格说明 v1.0 (毕业设计演示版)

- 文档版本：v1.0 (毕业设计演示版 / Defense-ready)
- 文档日期：2026-04-14
- 文档定位：定义 V1.0 版本的正式系统边界、模块职责与接口合同。
- 使用方式：作为系统验收、论文描述与答辩演示的正式依据。

## 1. 系统范围说明

本系统是一个面向《伤寒论》单书场景的研读支持系统，其 V1.0 正式运行边界为：

`提问 -> 混合检索 -> 证据分层 -> 回答编排 -> LLM 改写 -> 结果输出 (含历史记录)`

系统核心能力：
1. **语义理解**：支持现代汉语提问，利用向量技术跨越古今语义鸿沟。
2. **证据驱动**：所有回答必须基于书内真实条文，严禁脱离证据的自由生成。
3. **分层溯源**：提供清晰的证据等级划分与原文引用。
4. **会话管理**：支持多轮交互的历史持久化。

## 2. 正式模块清单 (V1.0)

### 2.1 离线构建模块
| 模块 ID | 模块名 | 责任 |
| --- | --- | --- |
| B1 | 数据预处理模块 | 清洗、结构化《伤寒论》文本，生成 Safe 数据包。 |
| B2 | 数据库构建模块 | 构建包含 FTS5 索引的 SQLite 运行库。 |
| B3 | 向量索引构建模块 | 生成基于 BGE 模型的 FAISS 稠密检索索引。 |

### 2.2 在线运行模块
| 模块 ID | 模块名 | 责任 |
| --- | --- | --- |
| R1 | 混合检索引擎 | 执行 FTS5 稀疏召回、FAISS 稠密召回与 RRF 融合。 |
| R2 | 证据门控模块 | 执行证据分层 (Primary/Secondary/Review) 与模式裁决。 |
| R3 | 回答编排模块 | 识别问题类型，根据证据槽位组织回答框架。 |
| R4 | Minimal API | 提供 `/api/v1/answers`、`/api/v1/conversations` 等接口。 |
| R5 | 聊天式前端 | 响应式 Web 界面，支持 Markdown、证据分区与历史列表。 |
| R6 | LLM 接入模块 | 对接阿里云 Model Studio，执行回答文本的生成与润色。 |
| R7 | 会话存储模块 | 管理 `chat_history_v1.db`，实现聊天记录持久化。 |

## 3. 关键接口合同

### 3.1 问答接口 (POST /api/v1/answers)
- **请求**：`{"query": "..."}`
- **响应**：包含 `answer_text`、`primary_evidence`、`citations`、`answer_mode` 等字段的标准化 Payload。

### 3.2 会话接口 (POST /api/v1/conversations)
- **功能**：创建新的会话记录。
- **关联**：后续消息将通过会话 ID 进行关联存储。

## 4. 证据分层协议 (Frozen)

| 层级 | 定义 | 典型内容 |
| --- | --- | --- |
| **Primary** | 核心证据 | 与查询直接相关的《伤寒论》原条文。 |
| **Secondary** | 辅助依据 | 相关联的条文、注解或补充说明。 |
| **Review** | 参考资料 | 低置信度匹配内容或背景风险提示。 |

*规则：若 Primary 槽位为空，系统必须返回 `weak` 模式回答或触发 `refuse` 拒答。*

## 5. 验收与回归基线

系统必须通过以下核心样例的回归测试：
1. **精确查询**：如“黄连汤方”，需返回 `strong` 模式及正确引用。
2. **模糊理解**：如语义解释类问题，需展示 LLM 润色后的准确回答。
3. **超纲拒答**：如非中医经典问题，需明确触发 `refuse` 模式。
