## 运行方式

启动命令：

```bash
python -m backend.api.minimal_api --host 127.0.0.1 --port 8000
```

页面入口：
- `GET /` (同源 SPA)

接口入口：
- `POST /api/v1/answers`
- `POST /api/v1/conversations`

## 核心交互规则 (V1.0)

1. **聊天流体验**: 用户提问后，系统以气泡形式追加回答。
2. **历史回溯**: 点击侧边栏历史条目，主面板能正确恢复该会话的消息流。
3. **证据分层**: 核心条文显示在“主依据”区，注解显示在“辅助依据”区，风险条文显示在“核对材料”区。
4. **实时反馈**: 提交查询时按钮禁用，并展示 Loading 状态。

## 典型样例验证 (Regression)

### 1. `黄连汤方的条文是什么？` (Strong)
- 验证点：回答文本、主依据区、引用区均需显示。
- 结果：页面渲染完整，核心条文 ID `ZJSHL-CH-010-P-0145` 等清晰可见。

### 2. `烧针益阳而损阴是什么意思？` (Weak)
- 验证点：显示黄色 `review_notice` 警告，主依据区隐藏，展示辅助资料。
- 结果：渲染逻辑正确触发弱回答提醒。

### 3. `书中有没有提到量子纠缠？` (Refuse)
- 验证点：显著展示拒答原因，隐藏所有证据区，显示改问建议。
- 结果：成功拦截非系统范围问题。

V1.0 版本的前端界面已完全对齐“聊天式交互”与“历史记录持久化”的设计目标，且在三类核心问答模式下表现稳定。页面展示内容与后端返回的 Answer Payload 保持高度一致。
