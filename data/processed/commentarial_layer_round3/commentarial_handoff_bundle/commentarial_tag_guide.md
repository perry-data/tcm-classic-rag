# commentarial tag guide

本文件说明第二轮离线精修后的 `commentary_functions` 标签体系。

## 标签设计原则

1. 面向后续问答路由，而不是单纯做学术分类。
2. 一个 unit 至少 1 个标签，必要时多个标签并存。
3. 优先结合结构信号与语义信号，不只做机械关键词匹配。
4. theme 单元与条文单元可共用同一标签体系，但侧重不同。

## 标签列表

### `passage_explanation`

- 定义：围绕具体条文、节选条文或一组条文进行释义、分解与逐段说明。
- 常见信号：逐条讲解；【讲解】；条文原文；这一条

### `pathogenesis`

- 定义：强调病机、成因、邪正变化、证候形成机制。
- 常见信号：病机；成因；邪；机转；所以然

### `formula_analysis`

- 定义：围绕方义、方药组成、加减思路、方证对应展开说明。
- 常见信号：方义；方剂；药味；桂枝汤方；大承气汤

### `therapeutic_method`

- 定义：突出治法、处置原则、汗吐下和解温清补等策略。
- 常见信号：治法；当用；宜；不可发汗；随证治之

### `comparison`

- 定义：对条文、方证、证候、不同解释路径进行对比辨析。
- 常见信号：鉴别；比较；不同；对比；相呼应

### `clinical_application`

- 定义：将理法方药落到临床运用、辨证思路、病机迁移上。
- 常见信号：临床；应用；治疗；现代临床；常用

### `clinical_case`

- 定义：引用或讨论具体医案、病例、案头经验。
- 常见信号：医案；病例；《续名医类案》；案

### `study_method`

- 定义：面向学习者的方法指导、读原文方法、教学法与学习要求。
- 常见信号：学习；方法；要求；怎样学；读经典

### `theory_overview`

- 定义：绪论、总论、六经辨证框架、概述性理论说明。
- 常见信号：绪论；概述；六经辨证；是一部什么书

### `edition_history`

- 定义：涉及作者、版本、沿革、底本、成书与流传史。
- 常见信号：版本；沿革；赵开美；成无己；注解伤寒论

### `textual_relation`

- 定义：说明上下条、前后条、互文、条文之间承接或索引关系。
- 常见信号：上条；下条；前后；互文；联系

### `commentator_views`

- 定义：突出讲者本人判断，或交代注家异同与个人取舍。
- 常见信号：我认为；我个人认为；有的注家；历代注家；有人认为

### `summary`

- 定义：对单元核心结论进行提要、归纳或概括。
- 常见信号：提要；总结；这一条论述；要点


## 赋标规则摘要

### 刘渡舟

- exact 单元默认至少有 `passage_explanation`
- 出现“病机/邪气/阴阳/机转”时加 `pathogenesis`
- 出现方后、药味、方证、具体汤方时加 `formula_analysis`
- 出现“宜/不可发汗/随证治之”等治法指向时加 `therapeutic_method`
- 出现“有的注家/我认为/我个人认为”时加 `commentator_views`
- 出现“合参/比较/鉴别/上条/下条/前后”时加 `comparison` 或 `textual_relation`

### 郝万山

- 有 `【讲解】` 的条文单元默认给 `passage_explanation`
- 有 `【提要】` 时优先加 `summary`
- 有 `【治法】` 时加 `therapeutic_method`
- 有 `【方义】` 或 `【方剂】` 时加 `formula_analysis`
- 含“怎样学原文/读经典/教学方法/学习要求”时加 `study_method`
- 绪论、版本、作者、沿革、六经框架等主题单元优先加 `theory_overview` 或 `edition_history`

## 注意事项

- `comparison` 与 `textual_relation` 不完全相同：前者强调辨析差异，后者强调条文承接或互文关系。
- `edition_history` 只用于作者、版本、沿革、底本等内容，不应用于普通条文解释。
- `summary` 不是兜底垃圾桶；只有明显提要/概括性内容才使用。
