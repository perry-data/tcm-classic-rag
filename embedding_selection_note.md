# Embedding Selection Note

## 推荐方案

推荐：

- Embedding：`BAAI/bge-small-zh-v1.5`
- Reranker：`BAAI/bge-reranker-base`

选择理由：

- 对中文语义检索较友好
- 对“现代问法 -> 古文条文”比纯 BM25 更有补益
- 模型大小相对克制，适合当前单书项目
- 本地化部署后没有持续 API 成本

## 成本判断

### 本地方案

- 费用：接近零 API 成本
- 代价：首次下载模型，之后主要消耗本地 CPU / 内存；若 PyTorch 支持 `mps`，rerank 可部分利用 Apple GPU
- 适合：毕设演示、离线调试、低持续成本方案

### API 方案

- 费用：建库和查询都持续付费
- 优势：不需要维护本地模型
- 风险：长期成本高于本地方案

### 当前建议

- 当前优先本地方案
- 若本地依赖安装受阻，再考虑 API embedding 作为备选

## 备选方案

备选 1：

- `BAAI/bge-base-zh-v1.5`

优点：

- 可能带来更强语义能力

缺点：

- 更重，更慢，更不利于当前最小落地

备选 2：

- 外部 API embedding + 本地 reranker

优点：

- 工程实现比较直

缺点：

- 成本持续发生

## 不推荐方向

- 直接选超大 embedding 模型
- 直接选超大多语言通用模型作为最小版本
- 在本轮引入训练或微调

这些方向对当前项目来说都过重，不利于“可运行、可验证、可演示”的最小闭环。
