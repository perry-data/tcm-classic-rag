# General Question Strategy Spec

## 一、什么是“总括类 / 泛问类问题”

本轮把下列问题视为“总括类 / 泛问类问题”：

- 用户问的不是某一条具体条文
- 也不是某个方名的原文 / 组成 / 出处
- 而是在问某个病、证、总类主题“总的该怎么处理 / 有哪些情况 / 有什么分支”

当前典型形式包括：

- `X 应该怎么办`
- `X 怎么处理`
- `X 如何应对 / 如何治疗`
- `X 有哪些情况`
- `X 有什么分类 / 分支 / 处理思路`

当前重点样例：

- `太阳病应该怎么办？`

## 二、当前支持的模式

当前只做最小支持，优先覆盖两类：

1. `management`

- 触发词：`应该怎么办 / 怎么处理 / 如何应对 / 如何治疗`
- 目标：给出 2～3 条“典型处理分支”式整理

2. `overview`

- 触发词：`有哪些情况 / 有什么分类 / 有什么分支 / 分情况`
- 目标：给出 2～4 条“分情况 / 分证候 / 分处理线索”式整理

当前优先支持的主题锚点是书内常见病证类主题，例如：

- `太阳病`
- `少阴病`
- `伤寒`

## 三、如何与“单条直答类问题”区分

当前采用“先排除、再识别”的最小规则：

1. 先排除已经单独支持的类型

- 比较问答先于总括问答识别
- 方名直问不进入总括路径

2. 再看是否出现总括触发词

- 没有 `怎么办 / 有哪些情况 / 分支` 等触发词，不进入总括路径

3. 再抽取主题锚点

- 去掉触发词后，剩余主题至少要能形成一个最小主题词
- 主题过短、只剩空泛词，不进入总括路径

4. 伪泛问继续按原路径走

例如：

- `黄连汤方应该怎么办？`

虽然表面上有“怎么办”，但主题仍是方名，不应被当成总括类问题。

## 四、最小执行流程

总括类问题当前走一条轻量补丁路径：

1. 识别问题类型并抽取 topic
2. 仍先跑现有 hybrid retrieval
3. 再用 topic 做一次更贴近主题的 hybrid retrieval
4. 在现有 safe `main_passages` 中做轻量 branch scan
5. 只按可解释的分支线索组织回答，不自由发挥
6. 继续遵守 evidence gating 与 citations 输出

当前 branch scan 只使用：

- `main_passages` 的 A/B 级材料做分支识别
- `secondary_evidence / review_materials` 做弱化补充

当前优先识别的分支线索：

- `某方主之 / 宜某方`
- `不可...`
- `名曰 / 名为`
- `自愈 / 则愈 / 当须`

## 五、strong / weak_with_review_notice / refuse 判定逻辑

### 1. strong

满足以下最小条件时判为 `strong`：

- 能识别为总括类问题
- 能从 safe `main_passages` 中组织出至少 2 条互不重复的分支
- 分支依据仍然是 A 级主条
- 回答能明确写出“这是总括性问题，并非只有一个固定答案”

此时：

- `primary_evidence` 仍然只放主条
- `citations` 继续可供前端消费
- `secondary_evidence` 只作补充，不参与主结论

### 2. weak_with_review_notice

满足以下情形之一时判为 `weak_with_review_notice`：

- 主题过宽，不能稳定组织出至少 2 条可靠主分支
- 只能整理出零散线索
- 只有部分相关材料，尚不足以构成完整“分情况整理”

此时：

- 不输出 `primary_evidence`
- 只输出部分分支线索或待核对线索
- `secondary_evidence / review_materials` 保留
- 明确提示“当前只能部分整理，需核对”

### 3. refuse

当以下条件同时成立时判为 `refuse`：

- 已识别为总括类问题
- 但既组织不出可靠分支
- 也没有足够的弱证据材料可供最小整理

此时：

- 不输出概括性答案
- 只给收窄问题的 follow-up 建议

## 六、当前实现的边界与限制

当前实现刻意保持最小，不做以下扩展：

- 不穷尽某一主题下的全部分支
- 不重做 retriever
- 不重做数据库结构
- 不新增 payload 顶层字段
- 不做多书、多对象复杂组织
- 不做知识图谱

当前已知限制包括：

- topic 抽取仍是启发式规则，不是完整语义解析
- branch scan 主要依赖条文里的显式触发词，不覆盖隐含分支
- weak 模式下的“待核对线索”仍偏保守
- 某些过宽主题会被降级到 weak，而不是强行给 strong 概括
- 结果更像“最小研读支持整理”，还不是“完整专题综述”
